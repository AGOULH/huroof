# لعبة الحروف — سياق المشروع (CLAUDE.md)

> ملف سياق لـ Claude Code. يحفظ في جذر المستودع ويُقرأ تلقائياً.

## نظرة عامة
**لعبة الحروف** — تطبيق ويب تعليمي/ترفيهي للأطفال (٥–١٢ سنة)، نسخة عربية بمحاذاة **RTL**.
الفكرة لعبة "اسم بنت / ولد / جماد / حيوان / طعام / بلد": يُختار حرف عشوائي ويكتب اللاعب كلمة تبدأ به لكل خانة ضمن وقت محدد.

التطبيق **PWA** (يُثبّت على الجوال) بملف واحد أساسي، يُنشر على **GitHub Pages** تحت الحساب `agoulh`.

## الملفات
- `index.html` — التطبيق كامل (HTML + CSS + JS في ملف واحد، ~570 سطر). كل المنطق هنا.
- `manifest.json` — إعدادات الـ PWA (اسم، أيقونة، standalone، RTL).
- `sw.js` — Service Worker، تخزين مؤقت cache-first (المفتاح `huroof-v2`). **غيّر رقم الإصدار عند أي تحديث** ليصل للمستخدمين.
- `icon.svg` — أيقونة التطبيق (تدرّج + حرف الألف، vector مستقل عن الخطوط).
- `test_full.js` — اختبار jsdom (اختياري، للتطوير). يفحص المعرّفات + منطق الإجابات + المؤقّت المتزامن + تدفّق اللعب.

## التقنيات والقيود
- **Vanilla JS** فقط (IIFE)، بدون أطر عمل ولا خطوة بناء.
- خطوط: **Lalezar** (عناوين) + **Tajawal** (نصوص) من Google Fonts، مع بدائل نظام.
- **Firebase Firestore** (compat v10.12.2 عبر CDN) — لوضع التحدي بين الأجهزة فقط.
- تصميم **mobile-first** (عرض أقصى 480px)، أزرار كبيرة، ألوان مبهجة.
- **قاعدة مهمة عند التطوير:** حافظ على **ملف واحد**، **RTL** (`dir="rtl"`)، وبدون مكتبات ثقيلة.

## آلية اللعب والاحتساب
- البيانات في أعلى الـ `<script>`: `ANIMALS` (١٢ شخصية إيموجي)، `LETTERS` (٢٣ حرف عربي سهل)، `CATS` (٦ خانات).
- **قاعدة الاحتساب (مهمة):** دالة `matches(input, letter)` = `isWord(input) && startsWithLetter(input, letter)` — الإجابة صحيحة إذا **بدأت بنفس الحرف** وكانت **كلمة من حرفين فأكثر**، **بدون أي قاموس أو تحقق من معنى الكلمة**. أي اسم/كلمة يكتبها الطفل تُقبل ما دامت تبدأ بالحرف.
  - `startsWithLetter` تطبّع أشكال الألف (`أإآٱ → ا`)، و`ى→ي`، `ة→ه`، `ؤ→و`، `ئ→ي`، وتتجاهل التشكيل، وتقبل الحرف بعد "ال".
  - `isWord` ترفض كتابة **الحرف وحده** (يجب أن يبقى حرفان على الأقل بعد تجاهل "ال" التعريف)، فلا يُحتسب مثل `ب` أو `الب`.
- كل إجابة صحيحة = **⭐ + ١٠ نقاط** (المجموع 60).
- مؤقّت لكل خانة، ثم انتقال للخانة التالية.

## الواجهات (Screens)
كل واجهة `<section class="screen" id="...">` وتُفعّل بإضافة `.active` عبر `show(id)`:
`screen-start` ← `screen-lobby` (تحدي) ← `screen-reveal` (فردي) ← `screen-countin` (تحدي) ← `screen-play` ← `screen-results`.

## الوضعان
### ١) فردي (Solo) — يعمل بدون أي إعداد
`startSolo()` → `beginCategorySolo()` (مؤقّت 60ث عبر `setInterval`) → `endCategorySolo()` → `showResultsSolo()`.
زر "التالي" متاح للتقديم اليدوي. سجل أبطال محلي في `localStorage` (`huroof_scores_v1`).

### ٢) تحدي بين الأجهزة (Multiplayer) — يحتاج Firebase
- نموذج البيانات في Firestore:
  - `rooms/{رمز ٤ أرقام}`: `{ host, status: "lobby"|"playing", round, perCat, letter, startedAt(ms), createdAt }`
  - `rooms/{code}/players/{playerId}`: `{ name, animal, score, stars, done, joinedAt }`
- `createRoom()` / `joinRoom()` + مستمعات `onRoomSnap`, `onPlayersSnap`.
- البدء: المضيف يضبط `startedAt = Date.now() + 3000` و`status="playing"` و`round++`.
- **المزامنة بدون رسائل لكل ثانية:** `mpCategoryState(elapsed, perCat, numCats)` يحسب لكل جهاز محلياً: العدّ التنازلي (elapsed سالب)، رقم الخانة الحالية، الوقت المتبقي، والانتهاء — كلها من `startedAt` المشترك. هذا يضمن تزامن الجميع.
- `runMP()` حلقة `setInterval(150ms)` مع `mpRunToken` لإلغاء الجولات القديمة. `finalizeMP()` يكتب النتيجة. `renderMpLeaderboard()` لوحة حية بالميداليات 🥇🥈🥉.
- المضيف يختار وقت الخانة: **30 أو 60 ثانية**.

## الأصوات
- مؤثرات Web Audio مولّدة برمجياً عبر `snd(name)` (لا ملفات صوت خارجية).
- نطق عربي محفّز عبر `speak(text)` (Web Speech API، صوت `ar-SA` إن توفّر، وإلا يتجاهل بهدوء).
- زرّان للتحكم: `soundBtn` (المؤثرات)، `voiceBtn` (النطق).

## إعداد Firebase (مرة واحدة لتفعيل التحدي)
1. أنشئ مشروع في `console.firebase.google.com`.
2. فعّل **Firestore Database** (وضع Test mode يكفي للعب العائلي).
3. أضف تطبيق **Web `</>`** وانسخ `firebaseConfig`.
4. الصق القيم مكان `PASTE_...` في كائن `FIREBASE_CONFIG` أعلى الـ `<script>` في `index.html`.

## قواعد التطوير لـ Claude Code
- لا تكسر نمط **الملف الواحد** ولا الـ **RTL**.
- بعد أي تعديل في `index.html`، **شغّل `node test_full.js`** قبل الدفع.
- إذا غيّرت أصول مخزّنة، **ارفع رقم إصدار الكاش في `sw.js`**.
- استخدم متغيرات الألوان في `:root` بدل ألوان ثابتة جديدة.
- أبقِ النصوص الموجّهة للأطفال بسيطة وعربية فصيحة/بيضاء.

## أفكار للتطوير القادم
- حفظ سجل الأبطال سحابياً (Firestore) بدل المحلي فقط.
- خانات/مستويات أصعب للأكبر سناً، أو اختيار عدد الخانات.
- اختيار الحرف يدوياً، أو استبعاد حروف صعبة.
- موسيقى خلفية + مزيد من المؤثرات.
- وضع المتفرّج، ونقل صلاحية المضيف عند خروجه.
- أيقونات PNG (192/512) لو احتجت نشر في متجر لاحقاً.
